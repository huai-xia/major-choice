# 学科门类总览

本指南涵盖的 8 个专业主要分布在工学门类下，少量涉及理学交叉。理解学科门类的区别，是选专业的第一步。

---

## 工学 vs 理学：核心区别

| | 工学 | 理学 |
|------|------|------|
| **核心目标** | 解决问题、建造系统 | 发现规律、建立理论 |
| **思维方式** | 工程思维：如何在约束下做出可用的东西 | 科学思维：为什么是这样？背后规律是什么？ |
| **典型课程** | 大量实验、课设、项目、实习 | 理论推导、证明、实验验证 |
| **就业导向** | 直接对口产业界 | 深造率高，科研/教育/产业均可能 |
| **授予学位** | 工学学士 | 理学学士 |

> **简单记忆**：理学回答"为什么"，工学回答"怎么做"。AI 领域两者都需要——数学/物理（理学）提供理论基础，CS/软工（工学）负责工程实现。

| 专业 | 所属门类 | 学位 | 说明 |
|------|---------|------|------|

| 机器人工程 | 工学 | 工学学士 | 交叉学科，融合机械 + 控制 + AI |

## 这些专业之间的"亲缘关系"

```
              数学/物理 (理学基础)
                    │
    ┌───────┴───────┬───────┴───────┬───────┐
    │               │               │       │
  微电子          计算机科学       自动化   机器人工程
 (芯片硬件)      (软件核心)      (智能控制) (具身智能)
    │          ┌───┼───┬───┐       │
    │          │   │   │   │       │
    │        AI  数据 软工 网安     │
    │          │   │   │   │       │
    └──────────┴───┴───┴───┴───────┘
              (中游核心技术)
```

- **计算机科学与技术**是其他所有专业的"母专业"——转任何 CS 方向都相对容易
- **人工智能**、**数据科学**、**软件工程**、**网络空间安全**都是从 CS 分化出来的，核心基础课大量重叠
- **微电子**最独立，偏硬件和物理，和 CS 交集较少但依赖关系强（AI 算法需要芯片算力）
- **自动化**和**机器人工程**是 CS + 硬件的交叉区域，两者互为近亲

## 选门类的实用建议

### 选工学的情况
- 你**喜欢动手**，不满足于纸上推导
- 目标明确：**毕业后进入产业界**
- 享受从 0 到 1 建造一个能工作的系统

### 选理学的情况
- 你对**底层原理**有强烈好奇心
- 不排斥**长期深造**（读研读博）
- 想做**算法研究员**而非算法工程师

> **对于 AI 领域**：工学路线更稳妥（就业面宽、容错率高），理学路线更适合有学术追求的学生。如果你还不确定，工学通常是更安全的选择。
